pelican 发布

生成

生成站点

使用缺省的theme产生一个site。

pelican /path/to/your/content/ [-o /path/to/your/output] [-s path/to/your/settings.py]

如果从quickstart配置缺省的content 生成到 output，并使用 publishconf.py，

$ pelican content -o output -s publishconf.py

可缩减为

$ pelican content

生成部分文件

也可对单篇文章或者页面进行生成操作。需要注意：该命令需要制定输出的文件，也可指定多个文件。详情可参考Writing only selected content

$ pelican --write-selected output/posts/my-post-title.html

自动生成

如果你觉得每次修改后手工启动生成太麻烦，可以设置pelican为监视模式，每次修改后，pelican会自动重新生成。

$ pelican --autoreload （或者是 -r）

对于非windows系统，可以在自动生成的同时，将output放到 http://localhost:8000 服务上。

$ pelican --autoreload --listen

查看产生的文件

直接用浏览器查看

由于是静态站点，所以可以用浏览器直接查看output中的文件。但是部分链接文件（如css，以及链接的其他文件）会有问题。

内建server

pelican提供了一个内建的server，通过 http://localhost:8000/ 可以查看相关文件。(windows系统异常)

$ pelican --listen

部署

生成

在部署前，可以使用publishconf.py重新生成整个网站

$ pelican content -s publishconf.py

可以通过 from pelicanconf import *,在 publishconf.py 中导入 pelicanconf 的配置。使用 pelican-quickstart 产生的 publishconf.py 中包含此行语句。

部署到SSH服务器

如果你通过SSH访问Nginx或者Apache服务器，你可以使用 rsync 来传送站点文件。

rsync -avc --delete output/ host.example.com:/var/www/your-site/

部署到 github

将output中的文件导出到master分支中

$ ghp-import -m "Generate Pelican site" --no-jekyll -b master output

将本地内容推送到github即可访问相关网页

$ git add content
$ git commit -m 'added a first post, a photo and an about page' $ git push origin content

访问

https://username.github.io

自动化

invoke

利用在quickstart生成的makefile和task.py，使用invoke可以简化上述流程。 首先需要单独安装 invoke (可能需要sudo权限)

$ pip install invoke

如果有特殊需要，可以修改tasks.py。然后生成站点：

$ invoke build

如果希望每次改动后自动重建站点，就使用下面的命令：

$ invoke regenerate

启动本地浏览服务 http://localhost:8000/:

$ invoke serve

每次修改后自动重载浏览服务，需要先安装 pip install livereload，然后使用如下命令：

$ invoke livereload

SSH 部署

$ invoke publish

make

在POSIX系统下，利用make命令，结合makefile，也可以实现自动化（类似invoke）
